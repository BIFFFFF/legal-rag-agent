# 法律报告与文书 Skill 目录

这个文件是 skill 总目录，只用于告诉 Agent 当前有哪些报告/文书生成能力、每个能力能做什么、对应读取哪个具体 skill 文件。

当用户明确提出“生成报告”“保存 Markdown”“生成 md 文件”“写文书”“起草合同/借条/函件”等需求时，Agent 应先读取本目录，再读取对应的具体 skill 文件。

## 可用 Skill

| Skill 名称 | 对应文件 | 主要能力 |
| --- | --- | --- |
| 法律咨询分析报告 | legal_consultation_report.md | 把普通法律问题整理成可保存的法律分析报告 |
| 合同审查报告 | contract_review_report.md | 审查合同条款，输出风险等级、修改建议和替代条款 |
| 借贷纠纷分析报告 | loan_dispute_report.md | 分析借条、欠条、转账、利息、还款、诉讼风险 |
| 判决书摘要报告 | judgment_summary_report.md | 提炼判决书事实、争议焦点、裁判理由和裁判结果 |
| 证据清单报告 | evidence_list_report.md | 整理已有证据、证明目的、缺失证据和补充取证建议 |
| 借条草稿 | borrow_note_draft.md | 起草借条模板，补充借款金额、利息、期限、签名等条款 |
| 还款协议草稿 | repayment_agreement_draft.md | 起草债务确认、分期还款、逾期责任等协议条款 |
| 租赁合同草稿 | lease_contract_draft.md | 起草简版房屋租赁合同，覆盖租金、押金、维修、解除等事项 |
| 催款函草稿 | demand_letter_draft.md | 起草正式、克制、可发送的催款函或履约催告函 |
| 解除合同通知书草稿 | termination_notice_draft.md | 起草解除合同通知，说明解除事实、依据和后续处理要求 |

## 使用规则

- 普通法律问答不需要生成报告，也不需要保存文件。
- 用户要求生成报告、保存 md、写文书、起草文书时，才使用这些 skill。
- 读取本目录后，应根据用户意图选择一个最匹配的具体 skill 文件。
- 若用户意图不清楚，应先说明可选报告/文书类型，并让用户选择。
- 若基于材料生成报告，应先读取材料，再检索法律依据，最后按 skill 模板生成。
- 生成的 Markdown 文件统一保存到 `output/` 文件夹。
- 文件名使用中文，例如 `借贷纠纷分析报告.md`、`合同审查报告.md`、`借条草稿.md`。
