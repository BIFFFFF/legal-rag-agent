# 判决书摘要报告 Skill

## 适用场景

用户提供判决书、裁定书、调解书等司法文书，并要求总结案件、提炼争议焦点或形成摘要报告。

典型请求：

- “帮我总结这个判决书”
- “提取法院认为部分”
- “整理争议焦点和裁判结果”
- “生成判决书摘要报告”

## 需要提取的信息

- 案件名称。
- 案号。
- 法院。
- 当事人。
- 案由。
- 原告诉讼请求。
- 被告答辩意见。
- 法院查明事实。
- 争议焦点。
- 裁判理由。
- 裁判结果。
- 引用法条。

## 生成步骤

1. 读取判决书材料。
2. 区分当事人主张、法院查明事实和法院裁判观点。
3. 提炼争议焦点。
4. 总结裁判理由和裁判结果。
5. 提取对用户有价值的启示。
6. 保存 Markdown 报告。

## 建议结构

```markdown
# 判决书摘要报告

## 一、案件基本信息

## 二、当事人诉辩意见

## 三、法院查明事实

## 四、争议焦点

## 五、裁判理由

## 六、裁判结果

## 七、引用依据

## 八、对用户的启示

## 九、免责声明
```

## 注意事项

- 不要把原告主张误写成法院认定。
- 不要把被告抗辩误写成事实。
- 材料不完整时，应提示摘要可能不完整。
- 裁判结果必须尽量准确，不要自行推断。
